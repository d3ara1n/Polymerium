## 下载

从 [GitHub Releases](https://github.com/d3ara1n/Polymerium/releases) 下载适合你平台的最新版本。

| 平台 | 文件 |
|----------|------|
| Windows x64 | `Polymerium-win-Setup.exe` |
| Linux x64 | `Polymerium.AvaloniaImage` |
| macOS ARM | `Polymerium-osx-Setup.pkg` |

每个平台使用其标准的安装程序格式。运行它并按照提示操作。

## macOS：PKG 安装提示

由于 PKG 安装包未经 Apple 开发者证书签名，macOS Gatekeeper 可能会阻止安装并提示「安装包已损坏」。解决方法：

1. 右键点击 `.pkg` 文件，选择 **打开**。
2. 如果仍然无法打开，在终端中移除隔离标记：

```bash
xattr -d com.apple.quarantine Polymerium-osx-arm64-Setup.pkg
```

3. 再次打开 `.pkg` 文件，按照安装向导操作即可。

> 这是一次性操作。后续版本可能会切换为 DMG 分发，届时将不再出现此问题。

## Windows：启用开发者模式

Polymerium 使用符号链接在实例之间共享模组文件，无需重复存储。在 Windows 上，创建符号链接需要启用 **开发者模式**。

1. 打开 **设置 → 系统 → 面向开发者**
2. 打开 **开发者模式** 开关
3. 确认提示

这是一次性设置。这不会影响安全性 — 它只是允许你的用户账号创建符号链接，这是 macOS 和 Linux 的默认行为。

> macOS 和 Linux 用户不需要任何特殊配置 — 符号链接开箱即用。

## 首次启动：首次启动向导

当你第一次打开 Polymerium 时，首次启动向导（OOBE）会引导你完成以下步骤：

1. **欢迎与功能** — 快速了解 Polymerium 能做什么。
2. **权限检查** — Polymerium 测试是否可以在其数据目录中创建符号链接。如果在 Windows 上失败，你需要启用开发者模式（见上文）。
3. **快速设置** — 可选地配置 Java 路径和语言。
4. **隐私** — 选择是否通过 Sentry 发送崩溃报告。
5. **完成** — 你准备好创建第一个实例了。

## 安装内容在哪里

从 **1.10.0** 开始，Polymerium 遵循各平台的标准目录约定，而不再把所有数据塞进同一个文件夹。数据按职责拆为四类，分别放在操作系统约定的位置：

- **配置（Config）** —— `settings.json`
- **数据（Data）** —— 持久化数据库（`persistence.sqlite.db`）与首次运行标记
- **缓存（Cache）** —— HTTP 缓存（`cache.sqlite.db`）、崩溃报告缓存，以及共享缓存：模组文件、游戏资源、Java 运行时（分别位于 `packages/`、`assets/`、`runtimes/`）
- **实例（Instances）** —— 你的 Minecraft 实例

| 类别 | Windows | Linux | macOS |
|---|---|---|---|
| 配置 | `%APPDATA%\Polymerium` | `~/.config/polymerium` | `~/Library/Application Support/dev.dearain.Polymerium` |
| 数据 | `%LOCALAPPDATA%\Polymerium` | `~/.local/share/polymerium` | （同配置） |
| 缓存（HTTP + 崩溃报告） | `%TEMP%\Polymerium` | `~/.cache/polymerium` | `~/Library/Caches/dev.dearain.Polymerium` |
| 缓存（共享：模组/资源/运行时） | `%TEMP%\Trident` | `~/.cache/trident` | `~/Library/Caches/dev.dearain.trident` |
| 实例 | `%LOCALAPPDATA%\Trident\instances` | `~/.local/share/trident/instances` | `~/Library/Application Support/dev.dearain.trident/instances` |

> **自 1.10.0 起的行为变化** —— 在 1.10.0 之前，Polymerium 在所有平台上都把**全部数据**统一存放在单个 `~/.trident/` 目录下（设置在 `~/.trident/.polymerium/`，共享缓存在 `~/.trident/cache/`，实例在 `~/.trident/instances/`）。出于兼容性考虑，如果旧版本已经创建了 `~/.trident/` 目录，Polymerium **会继续沿用该目录**，而不会改用上表中的平台分类位置——因此既有环境不会被动。全新的安装才会自动启用分类布局。
>
> 想把数据放到其他磁盘，或强制使用经典的单根目录布局？参见 [自定义数据目录 →](/docs/guides/custom-data-directory)。
