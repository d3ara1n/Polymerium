## 创建实例

1. 点击侧边栏或主区域中的 **+** 按钮。
2. 为你的实例命名（例如"我的第一个整合包"）。
3. 选择一个 **Minecraft 版本** — 例如 `1.21.4`。
4. 点击 **创建**。

就是这样。你的实例已经创建 — 但还没有下载任何游戏文件。

## 刚刚发生了什么？

在 Polymerium 中，实例不是一个装满文件的文件夹。它是一个 **元数据文件**（`profile.json`），描述了你的游戏应该包含什么：哪个 Minecraft 版本、哪个加载器、哪些模组。

实际的游戏目录是在你 **部署** 时按需构建的（当你第一次启动时会自动进行）。这就是 Polymerium 与传统启动器的不同之处 — 也是为什么切换加载器或稍后添加模组几乎是瞬间的。

> 你现在不需要理解这些细节。重要的是：**你的实例是一个描述，而不是副本**。如果你好奇，可以稍后阅读 [核心概念 → 元数据驱动的实例](/docs/concepts/metadata-driven)。

## 配置你的实例

创建实例后，在添加模组之前先配置它。打开实例并切换到 **设置** 标签页。

### 设置模组加载器

为你的实例选择一个模组加载器：

| 加载器 | 最适合 |
|--------|---------|
| **Fabric** | 轻量级、快速更新、大型性能模组生态系统 |
| **Quilt** | 兼容 Fabric，功能更多，开发更快 |
| **Forge** | 经典选择 — 旧版本最大的模组生态系统 |
| **NeoForge** | 现代 Forge 分支，许多 1.20+ 的新模组使用 |

你可以随时切换加载器 — 这是元数据驱动实例的优势。Polymerium 会在你下次部署时重建游戏目录。

### 内存分配

为这个实例覆盖最大内存（JVM 堆）。如果大型整合包需要比全局默认值更多的 RAM，这很有用。

### 命令包装器

为特殊情况设置命令包装器，例如具有双 GPU 的 Linux 系统上的 `prime-run`。包装器会作为前缀添加到 Java 启动命令中。

### 快速连接

设置一个服务器地址，在游戏启动时自动加入。无需每次在游戏中输入。
