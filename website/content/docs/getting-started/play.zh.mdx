## 部署你的实例

在 Minecraft 可以运行之前，Polymerium 需要 **部署** 你的实例。这是将你的元数据（版本、加载器、模组列表）转换为实际可运行的游戏目录的步骤。

点击 **部署** — 或者直接点击 **启动**，Polymerium 会在需要时自动部署。

### 部署做了什么

1. 将所有必需的文件 — Minecraft jar、加载器库和模组 — 下载到共享缓存
2. 使用指向缓存的 **符号链接** 构建游戏目录
3. 生成启动参数（classpath、JVM 参数等）

### 首次部署 vs 增量部署

首次部署会从头下载所有内容 — 预计时间与在任何启动器上安装新的整合包一样长。Polymerium 使用激进的优化下载策略，因此在良好的网络环境下，它可能比其他启动器更快。

真正的速度优势体现在后续部署中。当你做出小改动（添加模组、切换加载器版本）时，Polymerium 只下载新增或更改的内容，并使用符号链接重建游戏目录。增量部署很快 — 通常是近乎即时的。

### 快速模式

如果自上次部署以来没有任何更改（相同的模组、相同的版本、相同的设置），Polymerium 会 **完全跳过部署** 并直接进入启动。这使得后续启动感觉是瞬间的。

## 启动

点击 **启动** 来启动 Minecraft。Polymerium 支持三种启动模式：

| 模式 | 功能 |
|------|-------------|
| **托管**（默认） | Polymerium 跟踪游戏进程，捕获输出，并检测崩溃。推荐。 |
| **即发即忘** | 启动游戏并分离。如果你想在游戏时关闭 Polymerium，请使用此模式。 |
| **调试** | 导出完整的启动命令以供故障排除。实际上不启动游戏。 |
