## Creating an Instance

1. Click the **+** button in the sidebar or main area
2. Give your instance a name
3. Choose a **Minecraft version** (e.g. 1.21.4)
4. Select a **mod loader**:
   - **Fabric** — lightweight, fast updates
   - **Quilt** — Fabric-compatible, more features
   - **Forge** — the classic, massive mod ecosystem
   - **NeoForge** — modern Forge fork
5. Click **Create**

Your instance is created as a lightweight metadata file — no files are copied yet.

## Adding Mods

1. Select your instance in the sidebar
2. Switch to the **Mods** tab
3. Browse or search from **CurseForge** and **Modrinth** directly within Polymerium
4. Click **Install** on any mod to add it to your instance

Mods are stored in a shared cache. If another instance already has the same mod file, Polymerium simply links to it instead of downloading again.

## Deploying

Before launching, Polymerium needs to **deploy** your instance — this builds the actual game directory using symlinks from the shared cache.

1. Click **Deploy** (or it happens automatically on first launch)
2. Polymerium resolves all dependencies and creates the game directory
3. Click **Launch** to start Minecraft

Deployment is fast because there's no file copying — only symlink creation, which is nearly instantaneous.

## Managing Multiple Instances

Each instance is independent:

- Different Minecraft versions
- Different mod loaders
- Different mod sets
- Different settings

But they all share the same underlying mod files through symlinks. Adding a mod that's already cached in another instance takes zero extra disk space.
