## Installation

The `trident` CLI is bundled with Polymerium. On Windows, it's available as `Trident.Cli` in the application directory. You can also install it as a standalone .NET tool:

```bash
dotnet tool install --global Trident.Cli
```

## Instance Management

```bash
# Create a new instance
trident instance create my-pack --version 1.21.4 --loader fabric

# List all instances
trident instance list

# View instance details
trident instance inspect my-pack

# Deploy (build) an instance
trident instance build my-pack

# Launch an instance
trident instance run my-pack

# Delete an instance
trident instance delete my-pack
```

## Package Management

```bash
# Search for mods
trident package search "sodium"

# Add a mod by PURL (package URL)
trident package add modrinth:sodium
trident package add curseforge:jei

# List installed packages
trident package list

# Enable or disable a mod
trident package enable modrinth:sodium
trident package disable modrinth:sodium

# List available versions
trident package version list modrinth:sodium

# Pin to a specific version
trident package version set modrinth:sodium 0.6.0
```

## Export & Import

```bash
# Export as Modrinth modpack
trident instance export my-pack --format modrinth --output ./release/

# Import a modpack
trident instance import ./cool-pack.mrpack
```

## Account Management

```bash
# Add a Microsoft account (Device Code Flow)
trident account add

# List accounts
trident account list

# Remove an account
trident account remove <uuid>
```

## MCP Mode

`trident` can run as an MCP (Model Context Protocol) server, allowing AI agents like Claude, GPT, or Cursor to manage your Minecraft instances directly.

```bash
# Start MCP server (stdio transport)
trident --mcp
```

### MCP Configuration Example

Add to your AI tool's MCP configuration:

```json
{
  "mcpServers": {
    "polymerium": {
      "command": "trident",
      "args": ["--mcp"]
    }
  }
}
```

Once connected, your AI agent can:

- Create and manage instances
- Search and install mods
- Configure loaders and versions
- Export modpacks
- Manage accounts and repositories

### Example AI Prompts

- "Create a Fabric 1.21.4 instance named `performance-pack` with Sodium and Lithium"
- "Take a snapshot of my `skyblock` instance, then update all mods"
- "Export my `vanilla-plus` instance as a Modrinth modpack"

## Global Options

| Flag | Description |
|------|-------------|
| `--home <path>` | Specify data directory |
| `--json` | Output in JSON format |
| `--no-interactive` | Non-interactive mode (for scripts) |
| `--verbose` | Verbose logging |
| `--debug` | Debug logging |
