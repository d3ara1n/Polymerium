## 支持的格式

Polymerium 支持四种用于导入和导出的整合包格式：

| 格式 | 扩展名 | 说明 |
|--------|-----------|-------|
| **Trident**（原生） | `.zip` | 保留最多元数据：标签、规则、覆盖、图标、README、更新日志 |
| **CurseForge** | `.zip` | 行业标准；兼容 CurseForge 应用、Prism 等 |
| **Modrinth** | `.mrpack` | 现代格式；兼容 Modrinth 应用、Prism 等 |
| **MultiMC** | `.zip` | 兼容 MultiMC、Prism Launcher 及其衍生版本 |

## 导入整合包

1. 点击 **+** 按钮创建新实例。
2. 选择**从文件导入**。
3. 选择整合包存档。
4. Polymerium 自动检测格式并导入：

   - 元数据被解析并写入实例的 `profile.json`。
   - 整合包文件（配置、资源）被提取到 `import/`。
   - 整合包中列出的模组会被解析并下载到共享缓存中。

5. 部署并游玩。

> **更新整合包？**如果实例是通过导入创建的，您可以重新导入同一整合包的较新版本。Polymerium 会用新文件替换 `import/`，同时保持 `persist/` 不变 — 您移动到那里的文件将被保留。

## 导出实例

1. 选择一个实例。
2. 打开实例菜单 → **导出**。
3. 选择格式，填写元数据（名称、作者、版本、摘要）。
4. Polymerium 将实例元数据 + `import/` 文件打包为所选格式。

### 导出内容

| 内容 | 是否包含 |
|---------|:---------:|
| 模组列表 + 版本 | ✅ |
| 配置文件（`import/`） | ✅ |
| 实例图标 | ✅ |
| 整合包元数据（名称、作者、版本） | ✅ |
| 存档、截图、个人配置 | ❌ — 如果您将这些移动到 `persist/`，它们会保留在 `persist/` 中，不会包含在导出中 |

### 格式特定说明

- **Trident 格式**保留最多信息：标签、部署规则、账号绑定和额外文件。最适合 Polymerium 之间的共享。
- **CurseForge / Modrinth** 格式广泛兼容但会丢失 Polymerium 特定功能，如标签和规则。
- **MultiMC 格式**是最基础的 — 仅模组列表和配置。

## 在线与离线格式

并非所有整合包格式在底层的工作方式都相同：

- **CurseForge 和 Modrinth** 格式是**在线**格式 — 它们包含托管在这些平台上的模组的引用（包 URL），而不是实际的模组文件。导入时，启动器会解析这些引用并下载模组。

  这意味着 CurseForge 整合包理想情况下应仅引用 CurseForge 模组，Modrinth 整合包应仅引用 Modrinth 模组。如果您混合来源，整合包在发布时将无法通过平台审核。

- **Trident 和 MultiMC** 格式是**离线**格式 — 它们将实际模组文件打包在存档中。导入时无需互联网解析。

在为发布您的整合包选择格式时请记住这一点。
