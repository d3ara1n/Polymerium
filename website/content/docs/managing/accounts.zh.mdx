## 账号类型

Polymerium 支持四种账号类型：

| 类型 | 使用场景 | 认证方式 |
|------|----------|---------------|
| **微软** | 完整多人游戏、Realms、服务器 | 设备代码流程 → Xbox Live → Minecraft API |
| **离线** | 本地/单人游戏，无法访问服务器 | 仅用户名，生成确定性 UUID |
| **Authlib 注入器** | 第三方认证服务器（外置登录） | 使用 `-javaagent` 注入的 Yggdrasil 协议 |
| **试用** | 快速开始 — 无需登录即可试用启动器 | 具有固定角色的内置演示账号（无需任何微软账号） |

## 添加账号

进入侧边栏的**账号**页面。点击**添加账号**并选择类型：

### 微软账号

1. Polymerium 显示设备代码（例如 `A1B2C-3D4E5`）。
2. 在任意设备上打开 [microsoft.com/link](https://microsoft.com/link) 并输入代码。
3. 使用您的微软账号登录。
4. Polymerium 会自动检测登录并获取您的 Minecraft 个人资料。

您的令牌存储在本地，过期时会自动刷新。

### 离线账号

1. 输入用户名。
2. 就这样。Polymerium 根据名称生成确定性 UUID。

离线账号无法加入需要认证的服务器，但对于单人游戏和本地局域网完全适用。

> 离线账号仅在您添加微软账号后才可用。如果您只想在未登录的情况下试用启动器，请改用试用账号。

### 试用账号

试用账号预配置了固定角色 — 可直接上手游玩，无需任何设置。无需微软账号。

它们仅适用于本地和单人游戏，但在您决定完全登录前足够用来试用启动器。

### Authlib 注入器账号

1. 输入 Yggdrasil 认证服务器的**服务器 URL**（例如 `https://auth.example.com/api/yggdrasil`）。
2. Polymerium 获取服务器元数据，并在启动时注入相应的 `-javaagent` 参数。
3. 使用该服务器的凭据登录。

这对于使用第三方认证服务器（外置登录）的 Minecraft 社区非常有用。

## 按实例绑定账号

默认情况下，您选择的**默认账号**用于所有实例。但您可以为每个实例覆盖此设置：

1. 打开实例的属性。
2. 将**账号覆盖**设置为您已添加的任何账号。

这使您可以为不同服务器或游戏模式使用不同账号，而无需每次全局切换。

## 令牌刷新

- **微软账号**通过 OAuth 令牌刷新自动刷新 — 您无需重新登录。
- **Authlib 注入器账号**在每次启动前使用 Yggdrasil 协议进行验证和刷新。

如果令牌无法刷新（例如您撤销了访问权限），Polymerium 会提示您重新认证。

## 账号数据存储

所有账号数据存储在本地 SQLite 数据库中。凭据绝不会上传到任何地方。唯一的网络调用是向微软/Yggdrasil 认证服务器进行令牌操作。
