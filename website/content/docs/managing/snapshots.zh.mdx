---
title: 快照
description: 保存、恢复和比较您的整个游戏状态。自由实验 — 您随时可以回退。
---

## 什么是快照？

快照记录影响构建结果的实例所有内容：

- **实例配置** — 元数据（Minecraft 版本、加载器、模组列表、设置）
- **目录内容** — `import/` 和 `persist/` 中的所有内容，以及 `build/` 下 **import 的工作副本**

工作副本是来自 `import/` 的那些文件在运行时的状态——你玩过之后改过的配置等包内文件。它落在 `build/` 里。快照会记下这份状态，之后才能把这些修改找回来。

恢复快照时，元数据、`import/`、`persist/` 以及 `build/` 下的工作副本都会被写回。之后部署会从包和缓存补齐游戏还需要的其余部分。

## 何时使用快照

- **在进行风险更改之前** — 更新主要模组、切换加载器、调整配置
- **在进行实验之前** — 尝试可能想要撤销的新整合包配置
- **用于模组设置的"存档读档"** — 保留稳定设置的快照，以及一个用于实验的快照

## 创建快照

1. 选择您的实例。
2. 打开**快照**部分。
3. 点击**创建快照**。
4. 给它一个名称和可选的描述。

Polymerium 会记录当前的配置文件设置、`persist/` 与 `import/` 的内容，以及 `build/` 下的 import 工作副本。

## 恢复快照

1. 为您的实例打开**快照**部分。
2. 从列表中选择一个快照。
3. 点击**恢复**。

Polymerium 会将配置文件元数据、`import/`、`persist/` 以及 `build/` 下的工作副本恢复到拍摄快照时的状态。

## 比较快照（差异）

您可以比较两个快照以查看它们之间确切的变化：

- **元数据更改** — 不同的模组、版本或加载器设置
- **添加的文件** — `persist/`、`import/` 或 `build/` 工作副本中的新文件
- **删除的文件** — 在较早快照中存在但在较晚快照中不存在的文件
- **修改的文件** — 相同路径但内容不同

这对于了解模组更新或配置更改对实例的实际影响非常有用。

## 快照存储

快照存储实际的配置文件设置数据和目录文件内容 — 而不是对缓存的引用。这意味着快照是自包含的，即使共享缓存已被清除也可以恢复。

## 快照与备份

| | 快照 | 完整备份 |
|---|---------|------------|
| 存储内容 | 配置文件元数据 + 目录内容 | 完整文件副本 |
| 磁盘使用 | 紧凑（元数据 + 文件内容） | 与实例本身相同 |
| 速度 | 近乎即时 | 慢（复制所有内容） |
| 恢复 | 恢复元数据 + 目录（含工作副本） | 复制文件回来 |

快照比完整备份更轻量，创建速度更快。它们记录的是输入以及 import 的工作副本——不是 `build/` 下每一份日志、包链接或非托管文件——而这正是回到已知状态所需要的。
