---
title: 快照
description: 保存、恢复和比较您的整个游戏状态。自由实验 — 您随时可以回退。
---

## 什么是快照？

快照记录影响构建结果的实例所有内容：

- **实例配置** — 元数据（Minecraft 版本、加载器、模组列表、设置）
- **目录内容** — `import/`、`live/` 和 `persist/` 中的所有内容

快照不会直接修改 `build/`。相反，它们捕获输入以便稍后重现精确的游戏状态。恢复快照时，您的元数据和目录内容会被恢复 — 下次部署会从该状态重新构建 `build/`。

## 何时使用快照

- **在进行风险更改之前** — 更新主要模组、切换加载器、调整配置
- **在进行实验之前** — 尝试可能想要撤销的新整合包配置
- **用于模组设置的"存档读档"** — 保留稳定设置的快照，以及一个用于实验的快照

## 创建快照

1. 选择您的实例。
2. 打开**快照**部分。
3. 点击**创建快照**。
4. 给它一个名称和可选的描述。

Polymerium 会记录当前的配置文件设置以及 `persist/`、`import/` 和 `live/` 的内容。

## 恢复快照

1. 为您的实例打开**快照**部分。
2. 从列表中选择一个快照。
3. 点击**恢复**。

Polymerium 会将配置文件元数据和目录内容恢复到拍摄快照时的状态。您的游戏目录（`build/`）将在下次部署时重新构建以匹配该状态。

> 恢复快照不会立即修改 `build/`。下次部署将重现快照的状态。

## 比较快照（差异）

您可以比较两个快照以查看它们之间确切的变化：

- **元数据更改** — 不同的模组、版本或加载器设置
- **添加的文件** — `persist/`、`import/` 或 `live/` 中的新文件
- **删除的文件** — 在较早快照中存在但在较晚快照中不存在的文件
- **修改的文件** — 相同路径但内容不同

这对于了解模组更新或配置更改对实例的实际影响非常有用。

## 快照存储

快照存储实际的配置文件设置数据和目录文件内容 — 而不是对缓存的引用。这意味着快照是自包含的，即使共享缓存已被清除也可以恢复。

## 快照与备份

| | 快照 | 完整备份 |
|---|---------|------------|
| 存储内容 | 配置文件元数据 + 目录内容 | 完整文件副本 |
| 磁盘使用 | 紧凑（元数据 + 文件内容） | 与实例本身相同 |
| 速度 | 近乎即时 | 慢（复制所有内容） |
| 恢复 | 恢复元数据 + 目录，然后重新部署 | 复制文件回来 |

快照比完整备份更轻量，创建速度更快。它们捕获的是输入，而不是整个部署输出 — 这就是您所需要的，因为 `build/` 始终可以重建。
