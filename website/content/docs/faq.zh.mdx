## 为什么 Polymerium 在 Windows 上需要开发者模式？

Polymerium 使用符号链接（symlinks）在实例之间共享模组文件，而不是复制它们。在 Windows 上，创建符号链接需要开发者模式或管理员权限。启用开发者模式是一次性设置，只允许你的用户账户创建符号链接——它不会降低系统安全性。

macOS 和 Linux 原生支持符号链接，无需额外配置。

## 我的数据存储在哪里？

所有数据都保存在你的机器上：

- **设置**：私有数据目录 → `settings.json`
- **缓存与持久化**：私有数据目录 → `persistence.sqlite.db`
- **HTTP 缓存**：私有数据目录 → `cache.sqlite.db`
- **模组缓存**：共享缓存目录（可配置）

Polymerium 不上传任何数据。发送到 Sentry 的错误报告不包含个人信息。

## 可以从 Prism Launcher 或 MultiMC 迁移吗？

可以。从你当前的启动器以 Modrinth (.mrpack)、CurseForge (.zip) 或 MultiMC (.zip) 格式导出整合包，然后导入到 Polymerium 中。模组和配置都会保留。

## 支持哪些模组加载器？

Polymerium 支持四种加载器：

- **Fabric** — 轻量快速
- **Quilt** — 兼容 Fabric，功能更多
- **Forge** — 经典模组加载器
- **NeoForge** — 现代 Forge 分支

## Polymerium 免费吗？

是的。Polymerium 基于 MIT 协议开源，完全免费使用。没有广告、没有高级版、没有数据收集。

## Polymerium 支持哪些平台？

Polymerium 运行在：

- **Windows** x64
- **Linux** x64
- **macOS** ARM（Apple Silicon）

## 快照系统是如何工作的？

快照使用内容寻址存储来捕获实例的完整状态——模组、配置和设置。创建和恢复快照很快，因为它只记录引用，不复制文件。你还可以在快照之间进行差异对比，查看具体的变化。

## 可以在没有网络连接的情况下使用 Polymerium 吗？

可以。Polymerium 支持离线账户进行本地游戏。一旦模组被缓存且实例已部署，你可以在没有任何网络连接的情况下游玩。

## 什么是 MCP 模式？

MCP（模型上下文协议）是一种让 AI Agent 与外部工具交互的方式。Polymerium 的 `trident --mcp` 命令启动一个 MCP 服务器，暴露 30+ 个用于管理 Minecraft 实例的工具。这让你可以通过自然语言使用 Claude 或 GPT 等 AI 助手来创建实例、安装模组和管理游戏设置。

## 符号链接去重是如何工作的？

当你安装一个模组时，Polymerium 将实际文件存储一次到共享缓存中。当实例需要该模组时，Polymerium 创建一个指向缓存文件的符号链接，而不是复制它。游戏读取链接时就像读取原始文件一样——完全透明。

多个实例共享相同的模组文件都指向同一个缓存副本，所以 5 个包含 Sodium 的实例只占用 1 份的磁盘空间。
