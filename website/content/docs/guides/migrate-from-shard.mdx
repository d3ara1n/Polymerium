## Migration Path

Shard uses its own profile format and doesn't support modpack export. The migration path depends on what you have.

### If You Have the Original Modpack File

If you originally imported a CurseForge or Modrinth modpack into Shard:

1. Find the original `.zip` or `.mrpack` file.
2. Import it directly into Polymerium via **+ → Import from file**.
3. Set up your account and you're done.

### If You Built the Modpack in Shard

Shard doesn't have a standard export format that Polymerium can read. You'll need to recreate the modpack:

1. **In Shard**, note down your Minecraft version, loader, and mod list.
2. **In Polymerium**, create a new instance with the same version and loader.
3. **Add each mod** by searching CurseForge or Modrinth within Polymerium.
4. **Copy personal data** (saves, screenshots, options.txt) from Shard's instance directory to Polymerium's `persist/` directory. Create `.keep` files inside any directories you move — see [Four-Layer Directories](/docs/concepts/four-layers#persist---user-managed-persistence) for how this works.

### Shard Instance Location

Shard stores instances in:

| Platform | Path |
|----------|------|
| Windows | `%APPDATA%\shard-launcher\` |
| macOS | `~/Library/Application Support/shard-launcher/` |
| Linux | `~/.local/share/shard-launcher/` |

### Copying Saves and Settings

1. Find your Shard instance directory.
2. Copy `saves/`, `screenshots/`, `resourcepacks/`, `options.txt`.
3. Paste into your Polymerium instance's `persist/` directory. Create `.keep` files inside any directories you move.
4. Deploy — files appear in the game directory on next launch.

## Why Switch?

| Feature | Shard | Polymerium |
|---------|-------|------------|
| Windows symlink deduplication | ❌ Copies files | ✅ Real symlinks |
| Instance snapshots | ❌ | ✅ Save/restore/diff |
| Chinese localization | ❌ | ✅ |
| Offline accounts | ❌ | ✅ |
| Authlib Injector | ❌ | ✅ |
| Modpack export | ❌ Import only | ✅ 4 formats |
| MCP / AI Agent mode | ❌ | ✅ 30+ tools |
