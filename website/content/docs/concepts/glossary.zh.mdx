---
title: 术语表
description: 用直白语言解释 Polymerium 的部署、目录、包和运行时等概念。
---

Polymerium 的元数据驱动模型包含一些特有概念。本术语表用于统一产品用语，并说明这些词对用户实际意味着什么。

## 核心概念

| 术语 | 含义 |
|------|------|
| **实例** | 由 Polymerium 管理的一套 Minecraft 环境，包括实例配置和受管理的目录。 |
| **实例配置** | 定义实例的元数据，包括 Minecraft 版本、模组加载器、包、部署规则和设置。 |
| **整合包** | 可分发的一组包、配置和源文件。 |
| **包** | 指向资源仓库内容的受管理引用。包不只包括模组，也包括资源包、光影包、数据包和世界。 |
| **资产** | Polymerium 管理的实际文件，例如配置、截图、日志、库或版本文件。资产和资源仓库中的包不是同一个概念。 |
| **资源仓库** | 托管包及其版本的服务或来源，例如 Modrinth 或 CurseForge。 |
| **Pref（包引用）** | 标识资源仓库中某个包或版本的引用，通常表示为 `pref://` URI。 |

## 实例生命周期

| 术语 | 含义 |
|------|------|
| **部署** | 根据实例配置、包、整合包源和本地保留数据准备可运行的游戏目录。部署负责准备文件，不会启动 Minecraft。 |
| **启动** | 启动 Minecraft。启动前可能会自动部署，但启动和部署仍是不同操作。 |
| **重置** | 清空运行目录和部署锁，使 Polymerium 可以重新部署。除非数据已经移入本地保留，否则重置可能永久删除存档、截图、日志，以及游戏或模组生成的配置。 |
| **快照** | 包含实例配置和 Polymerium 受管理数据的可恢复状态。快照比完整备份轻量，但不会自动记录运行目录中的每一个非受管文件。 |
| **共享缓存** | 由多个实例复用的已下载包、Minecraft 资产、库和运行文件。缓存不是用户数据。 |

## 受管理目录

第一次提到目录时，应同时使用产品名称和实际文件夹名称。

| 术语 | 文件夹 | 含义 |
|------|--------|------|
| **整合包源** | `import/` | 整合包提供或准备导出的权威文件。整合包更新可能替换它；重置不会删除它。 |
| **本地保留** | `persist/` | 跨重置和整合包更新保留的用户数据。只有用户明确放入这里的数据才会受到保护。架构说明中也可称为“持久层”。 |
| **运行目录** | `build/` | Minecraft 实际读写的游戏目录，可能包含存档、截图、日志和生成的配置。重置会清空它。 |
| **工作副本** | `build/` 的一部分 | 整合包源文件部署后供游戏实际读写的副本。工作区会把它与整合包源进行比较。工作副本不等于整个运行目录。 |

## 工具与规则

| 术语 | 含义 |
|------|------|
| **工作区** | 比较整合包源与工作副本，并在两者之间同步选中改动的工具。 |
| **同步到 Import** | 把选中的工作副本改动同步到整合包源，使它成为整合包的一部分。 |
| **部署规则** | 改变匹配包部署方式的规则。 |
| **选择器** | 决定部署规则应用于哪些包的条件。 |
| **规则操作** | 选择器匹配后执行的效果，例如跳过或重定向包。 |

## 运行时与账号

| 术语 | 含义 |
|------|------|
| **Minecraft 版本** | 实例选择的 Minecraft 本体版本。 |
| **模组加载器** | Fabric、Forge、NeoForge、Quilt 或其他加载模组的系统。 |
| **加载器库** | 模组加载器运行所需的文件，不等于 Java 运行时。 |
| **Java 运行时** | 用于启动 Minecraft 的 Java 安装和可执行文件。 |
| **游戏账号** | 用于启动游戏的 Microsoft、离线、试用或外部验证账号。 |

## 数据安全原则

重置是修复操作，不是无害清理。只存在于运行目录中的重要内容都可能丢失。重置或进行彻底重建前，应先把存档、截图、自定义选项和其他个人数据移动到本地保留。
