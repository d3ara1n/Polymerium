> 制定日期：2026-07-14
> 定位：排查并优化实例安装页包数据加载期间的 UI 卡顿与异步工作调度
> 当前状态：未实施

## 计划

### 背景与动机

实例安装页在加载大量包信息时可能出现窗口短暂未响应。近期观察到缓存层日志洪水会显著放大这个问题，关闭高频缓存日志后体验明显改善，但这不等于页面加载链路本身已经没有卡顿风险。

该页面的包数据加载包含远端仓库请求、本地缓存访问、缩略图加载、包信息模型构建、列表过滤排序、分组与绑定通知。异步等待可以避免阻塞 I/O 等待时间，但不会自动把等待前后的数据整理、集合更新和 UI 绑定通知移出 UI 线程。后续需要对整条链路做一次专门排查，确认哪些工作真正需要在 UI 线程执行，哪些可以在后台完成，哪些通知需要合并或降频。

重点关注实例安装页当前由 InstanceSetupPageModel 驱动的包列表刷新路径，但不要把问题预设为某一个方法或某一个库；排查应覆盖从仓库数据进入到界面完成稳定渲染的完整用户可感知流程。

### 目标

- 明确实例安装页加载包信息时 UI 未响应的主要来源，区分 I/O 等待、CPU 数据处理、集合变更通知、图片解码、日志输出和控件渲染成本。
- 让包元数据与缩略图加载期间窗口保持可交互，滚动、切换筛选、展开折叠和导航不应因为后台加载而长时间停顿。
- 保持现有批量仓库查询与缓存命中收益，不因为优化 UI 响应性退化为大量逐项远端请求。
- 让后台准备的数据在进入 UI 前已经尽可能完整，UI 线程只承担必要的视图状态应用与绑定通知。
- 保证取消、页面离开、实例更新和重复刷新时不会把过期结果重新写回界面。

### 非目标

- 不重写仓库缓存架构，不替换 FusionCache 或 NeoSmart。
- 不改变包解析语义、依赖解析语义、规则匹配语义或实例数据格式。
- 不把所有页面的异步模式一次性重构；本轮以实例安装页和相同症状的直接共性为主。
- 不为了隐藏卡顿而移除必要的加载状态、错误提示或用户反馈。

### 关键决策与取舍

- 优先用测量结果决定优化点，而不是凭调用链猜测瓶颈。需要分别记录 UI 线程停顿、后台任务耗时、仓库请求耗时、缓存访问耗时和集合通知规模。
- 优先减少 UI 线程上的工作量和通知次数，而不是把现有逻辑粗暴包进后台线程。后台执行可以缓解部分问题，但如果结果仍以逐项通知风暴写回 UI，卡顿会转移到最后的应用阶段。
- 保留批量查询与批量缓存路径。响应性优化不能牺牲网络请求数量和缓存协调能力。
- 加载结果应以快照形式进入 UI。后台阶段准备稳定数据，UI 阶段集中应用，避免边加载边触发大量过滤、排序和重新布局。
- 取消与过期结果处理必须作为核心验收点。页面离开或实例数据变化后，旧加载任务不应覆盖新状态。

### 影响面

- 实例安装页的包列表刷新、包组信息加载、缩略图加载、筛选、排序和展开折叠体验。
- 仓库元数据读取链路，包括远端批量请求、本地缓存命中、缓存未命中后的回源和失败降级。
- 应用层图片缓存、缩略图解码与默认图片回退。
- 动态列表数据源、分组模型、标签筛选计数和结果计数等绑定通知。
- 页面生命周期、实例更新状态、取消令牌和重复刷新排队策略。
- 日志配置对 UI 响应性的影响，尤其是高频热路径日志在调试环境中的输出成本。

### 验收标准

- 打开包含大量包的实例安装页时，窗口不会因为包信息加载而进入长时间未响应状态。
- 加载期间用户可以滚动列表、切换筛选、展开折叠分组和离开页面，交互反馈不会被后台加载长时间阻塞。
- 缓存命中场景下不会产生大量日志输出，也不会因为读取大量缓存项导致 UI 明显卡顿。
- 缓存未命中场景下仍使用批量仓库查询，网络请求数量不随包数量线性暴涨。
- 页面离开、实例更新或再次触发刷新后，旧任务的结果不会覆盖新页面状态。
- 仓库慢、断网、图片下载失败或个别包解析失败时，页面能完成可用渲染并给出合理反馈。
- 优化后包列表中的项目数量、分组、筛选结果、包信息和缩略图与优化前保持一致。

### 备选方案备案

- 全面降低全局日志级别：可以减少部分输出，但会影响其他组件排障，并且不能消除特定库热路径日志的内部成本，因此只适合作为辅助开关，不作为主要方案。
- 把整个页面刷新包进后台线程：实现简单，但容易把 UI 对象、绑定集合和页面生命周期带入后台线程，增加线程安全风险，也无法解决最终写回 UI 时的通知风暴。
- 禁用部分列表功能换取流畅度：可以减少过滤排序和分组成本，但会直接削弱实例管理体验，除非测量证明某个功能成本极高且没有可接受的优化路径，否则不采纳。
- 放弃批量仓库请求改为逐项懒加载：首屏可能更早可见，但远端请求数量和缓存协调成本会上升，整体加载时间与服务端压力可能变差，不作为默认方向。
- 仅依赖虚拟化控件缓解渲染：虚拟化能降低可视区域渲染成本，但不能解决后台结果逐项写回、过滤排序重算和日志输出导致的主线程停顿，只能作为组合优化的一部分。
