# plans/

任务计划文件，彼此独立。本目录不维护索引、依赖图或清单——随时新建、随时归档，维护成本趋近于零。

一份文件在不同阶段是两种截然不同的东西：**实施前是构想（idea），实施后才是方案（真相）**。同一份文件，性质取决于它走到了哪一步。

## 流程

1. **写构想** — 想到要做什么时落盘。这是头脑风暴，只写想要什么：做什么、期望什么效果、调研得知的注意事项。不写怎么做、不写代码。构想可能全错，它只是起点，只能参考。
2. **实施** — 挑时间让 agent 读构想、理解意图，再对照当下真实代码临场决定 HOW。**实施过程中不回写本文件**——边做边写会让它停在"半构想半方案"的混沌态，既不是纯粹的 idea，也不是核实过的真相。
3. **回填方案** — 实施完成且验收后，把文件**整体替换**为实际怎么做、结果如何。从此这份文件才可信；实施前的构想原文不留——它只是想法，已经过时。

## 构想只写意图

构想只记三样：**做什么、想要什么效果、调研得知的注意事项**。其余不写——不写实现机制、不写代码、不写文件清单、不写进度。没确定的事宁可留白，绝不能凭推测写成言之凿凿的结论；构想里只要混入一个不符合现状的描述，接手者就会把整份判为不可信而弃用。

## 归档

`mv` 进 `archived/` 即代表 done。archived/ 是坟墓，归档物没有参考价值，不要读。
